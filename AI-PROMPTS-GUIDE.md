## 1. 開始專案時的提示詞

### 🟢 標準開始提示詞

```
我需要開始 TaskFlow Pro 專案的工作。

請按照以下順序執行：
1. 閱讀 AI-MANDATORY-CHECKLIST.md（強制檢查清單）
2. 閱讀 WORK_LOG_CURRENT.md 前 50 行（獲取最新狀態）
3. 閱讀 PROJECT-QUICKSTART.md（了解專案架構）
4. 閱讀 PROJECT-COMPREHENSIVE-SUMMARY.md（了解專案全貌）

然後回答以下問題：
- 當前前端生產環境 Deploy ID 是什麼？
- 當前後端 Docker 映像版本是什麼？
- 當前容器 ID 是什麼？
- 最後更新時間是什麼時候？
- 5 個絕對禁止的操作是什麼？

確認完成後，告訴我你準備好了，我會說明今天的任務。
```

### 🟢 快速開始提示詞（已熟悉專案）

```
開始 TaskFlow Pro 工作。

快速確認：
- 閱讀 WORK_LOG_CURRENT.md 前 50 行
- 確認當前版本號
- 確認核心規則

然後告訴我當前系統狀態，我會說明任務。
```

## 2. 修改代碼時的提示詞

### 🔧 前端修改提示詞

```
我需要修改前端代碼：[描述修改內容]

請遵循以下流程：
1. 確認當前前端 Deploy ID
2. 修改代碼
3. 清除舊構建：Remove-Item -Recurse -Force dist
4. 重新構建：npm run build
5. 部署到生產環境（使用正確的 Netlify Site ID）
6. Git commit 所有變更
7. 更新 WORK_LOG_CURRENT.md

注意事項：
- ✅ 必須清除 dist 資料夾
- ✅ 使用正確的 Netlify Site ID: 5bb6a0c9-3186-4d11-b9be-07bdce7bf186
- ✅ 記錄新的 Deploy ID
- ❌ 不要跳過任何步驟
```

### 🔧 後端修改提示詞

```
我需要修改後端代碼：[描述修改內容]

請遵循以下流程：
1. 確認當前後端 Docker 映像版本
2. 創建快照備份（使用 /root/create-snapshot.sh）
3. 創建修復腳本（Pure ASCII，中文使用 Unicode Escape）
4. 使用 Get-Content | ssh 管道上傳腳本
5. 在容器內執行修復腳本
6. 重啟容器：docker restart taskflow-pro
7. Commit 新映像：docker commit taskflow-pro taskflow-pro:v新版本號
8. 創建最終快照
9. Git commit 所有變更
10. 更新 WORK_LOG_CURRENT.md

注意事項：
- ✅ 修改前必須創建快照
- ✅ 修改後必須 docker commit
- ✅ 使用 Get-Content | ssh 管道（不要用 scp）
- ✅ PowerShell 使用 ; 分隔命令（不要用 &&）
- ✅ 路由文件必須 Pure ASCII
- ❌ 不要在容器運行時 docker commit
- ❌ 不要編譯 TypeScript（直接修改 JS）
```

### 🔧 資料庫修改提示詞

```
我需要修改資料庫：[描述修改內容]

請遵循以下流程：
1. 確認當前系統狀態
2. 創建快照備份（包含資料庫）
3. 創建資料庫修改腳本
4. 在容器內執行腳本
5. 驗證修改結果
6. 重啟容器
7. Commit 新映像
8. 創建最終快照
9. Git commit 所有變更
10. 更新 WORK_LOG_CURRENT.md

注意事項：
- ✅ 資料庫操作必須透過 dbCall 適配器
- ✅ 修改前必須備份
- ✅ 驗證修改結果
- ❌ 不要直接操作資料庫文件
```

### 🔧 同時修改前後端提示詞

```
我需要同時修改前後端：[描述修改內容]

請遵循以下流程：
1. 確認當前系統狀態
2. 先修改後端（按照後端修改流程）
3. 再修改前端（按照前端修改流程）
4. 測試前後端整合
5. Git commit 所有變更
6. 更新 WORK_LOG_CURRENT.md

注意事項：
- ✅ 先後端後前端
- ✅ 每個步驟都要完整執行
- ✅ 測試整合功能
- ❌ 不要同時修改（容易出錯）
```

## 3. 結束工作時的提示詞

### 🏁 標準結束提示詞

```
今天的工作已完成，請整理並總結。

請執行以下操作：
1. 檢查所有變更是否已 Git commit
2. 更新 WORK_LOG_CURRENT.md（如果還沒更新）
3. 更新記憶倉庫（如果有重要經驗）
4. 創建今日工作總結，包括：
   - 完成的任務列表
   - 修改的文件列表
   - 新的版本號
   - 遇到的問題與解決方案
   - 需要注意的事項

然後告訴我：
- 今天完成了什麼
- 當前系統狀態
- 下次工作的建議
```

### 🏁 快速結束提示詞

```
工作完成，快速總結。

請告訴我：
- 完成的任務
- 新的版本號
- Git commit 狀態
- 是否需要更新文檔
```

## 4. 緊急情況提示詞

### 🚨 系統故障提示詞

```
系統出現故障：[描述問題]

請立即執行以下診斷：
1. 檢查容器狀態：docker ps
2. 檢查容器日誌：docker logs taskflow-pro --tail 100
3. 檢查最近的修改（WORK_LOG_CURRENT.md）
4. 識別問題根源
5. 提供修復方案

不要立即修復，先告訴我診斷結果和建議的修復方案。
```

### 🚨 回滾提示詞

```
需要回滾到之前的版本。

請執行以下操作：
1. 列出最近 5 個快照
2. 確認要回滾的版本
3. 停止當前容器
4. 恢復快照
5. 啟動容器
6. 驗證系統狀態
7. 更新 WORK_LOG_CURRENT.md

等待我確認要回滾的版本後再執行。
```

### 🚨 資料恢復提示詞

```
需要恢復資料：[描述需要恢復的資料]

請執行以下操作：
1. 列出可用的備份
2. 確認備份時間點
3. 創建當前快照（以防萬一）
4. 恢復資料
5. 驗證資料完整性
6. 更新 WORK_LOG_CURRENT.md

等待我確認備份版本後再執行。
```

| 情況 | 使用提示詞 |
|------|----------|
| 開始新的工作會話 | 標準開始提示詞 |
| 已經熟悉專案，快速開始 | 快速開始提示詞 |
| 需要修改前端代碼 | 前端修改提示詞 |
| 需要修改後端代碼 | 後端修改提示詞 |
| 需要修改資料庫 | 資料庫修改提示詞 |

| 系統出現故障 | 系統故障提示詞 |

| 對話超過 20 輪 | 完整重置提示詞 |
| AI 開始違反規則 | 規則確認提示詞 |
| 需要深度診斷 | 深度診斷提示詞 |
| 需要更新文檔 | 相應的文檔更新提示詞 |

## ⚠️ 重要提醒

### 每個提示詞都應該包含的元素

1. **明確的目標**：清楚說明要做什麼
2. **具體的步驟**：列出執行步驟
3. **注意事項**：提醒禁止和必須的操作
4. **驗證要求**：說明如何驗證結果

### 使用提示詞的最佳實踐

1. ✅ **開始時總是使用開始提示詞**
2. ✅ **修改時總是使用相應的修改提示詞**
3. ✅ **結束時總是使用結束提示詞**
4. ✅ **長對話時定期使用重置提示詞**
5. ✅ **出現問題時使用緊急情況提示詞**

### 不要做的事

1. ❌ 不要跳過開始提示詞直接開始工作
2. ❌ 不要使用模糊的指令（如「修改一下」）
3. ❌ 不要在沒有確認的情況下執行危險操作
4. ❌ 不要忘記使用結束提示詞總結工作

## 🎯 提示詞效果評估

### 好的提示詞應該讓 AI：

- ✅ 清楚知道要做什麼
- ✅ 按照正確的流程執行
- ✅ 遵守所有規則
- ✅ 提供完整的結果
- ✅ 更新相關文檔

### 如果 AI 沒有按照預期執行：

1. 檢查提示詞是否清晰
2. 檢查 AI 是否閱讀了必要文檔
3. 使用重置提示詞重新開始
4. 如果仍然有問題，更新提示詞
