**日期**: 2026-01-06

## ⚠️ 重要提示

您看到的錯誤訊息：
```
WebSocket connection to 'wss://165.227.147.40:3000/ws' failed
```

這是因為瀏覽器不信任後端的自簽名 SSL 證書。

---

## 🔧 解決步驟（只需操作一次）

### 步驟 1：開啟新分頁

在瀏覽器中開啟新分頁，訪問：

```
https://165.227.147.40:3000
```

**重要**: 必須使用 `https://` 不是 `http://`

---

### 步驟 2：接受安全警告

瀏覽器會顯示類似以下的警告訊息：

#### Chrome 瀏覽器
1. 看到 "您的連線不是私人連線" 警告
2. 點擊 **"進階"**
3. 點擊 **"繼續前往 165.227.147.40 (不安全)"**

#### Firefox 瀏覽器
1. 看到 "警告：潛在的安全性風險" 警告
2. 點擊 **"進階"**
3. 點擊 **"接受風險並繼續"**

#### Edge 瀏覽器
1. 看到 "您的連線不是私人連線" 警告
2. 點擊 **"進階"**
3. 點擊 **"繼續前往 165.227.147.40 (不安全)"**

---

### 步驟 3：確認成功

接受證書後，您應該會看到類似以下的 JSON 回應：

```json
{
  "version": "3.9.0",
  "name": "TaskFlow Pro",
  "build": "2026-01-06T...",
  "status": "running"
}
```

看到這個回應表示證書已被接受。

---

### 步驟 4：關閉該分頁

關閉剛才開啟的分頁（顯示 JSON 的那個）。

---

### 步驟 5：重新整理主頁面

回到 TaskFlow Pro 主頁面，按 **F5** 重新整理頁面。

---

## ✅ 驗證連接成功

重新整理後，按 **F12** 打開開發者工具，查看 Console：

### 成功的訊息
```
[WebSocket] 連接到: wss://165.227.147.40:3000/ws
[WebSocket] 已連接
```

### 如果還是失敗
```

```

如果還是看到失敗訊息，請：
1. 確認您已經訪問了 `https://165.227.147.40:3000`（不是 http）
2. 確認您點擊了 "繼續前往" 或 "接受風險"
3. 確認您看到了 JSON 回應
4. 嘗試清除瀏覽器緩存後再試一次

## 📝 注意事項

1. **這是一次性操作** - 證書接受後會被瀏覽器記住
2. **每個瀏覽器獨立** - 如果您使用多個瀏覽器，每個都需要操作一次
3. **無痕模式** - 無痕模式下每次都需要重新接受證書
4. **安全性** - 這是測試環境的自簽名證書，生產環境應使用有效證書

---

## 🔐 為什麼需要這個步驟？

1. **後端使用自簽名證書** - 為了支援 HTTPS/WSS
2. **瀏覽器安全策略** - 預設不信任自簽名證書
3. **HTTPS 網站限制** - Netlify 使用 HTTPS，只能連接到 WSS（安全的 WebSocket）
4. **手動信任** - 瀏覽器要求用戶手動確認信任自簽名證書

---

## 🚀 長期解決方案

如果您不想每次都需要用戶手動操作，可以考慮：

1. **申請 Let's Encrypt 證書**（推薦）
   - 免費的有效 SSL 證書
   - 瀏覽器自動信任
   - 需要域名

2. **使用 Cloudflare Tunnel**
   - 提供有效的 HTTPS
   - 無需域名
   - 免費方案可用

3. **更換部署平台**
   - 使用支援 WebSocket 的平台
   - 例如：Railway、Render
