# WebSocket 連接設定指南

**日期**: 2026-01-06  
**版本**: v8.9.19

---

## ⚠️ 重要提示

由於後端使用自簽名 SSL 證書，瀏覽器會阻止 WebSocket (WSS) 連接。

---

## 🔧 解決方法

### 方法 1：接受自簽名證書（推薦）

1. **開啟新分頁**，訪問後端 HTTPS 地址：
   ```
   https://165.227.147.40:3000
   ```

2. **瀏覽器會顯示安全警告**，點擊：
   - Chrome: "進階" → "繼續前往 165.227.147.40 (不安全)"
   - Firefox: "進階" → "接受風險並繼續"
   - Edge: "進階" → "繼續前往 165.227.147.40 (不安全)"

3. **看到 API 回應後**，關閉該分頁

4. **重新整理主頁面**，WebSocket 應該可以正常連接

---

### 方法 2：使用環境變數（開發環境）

如果你在本地開發，可以設定環境變數：

```bash
# .env.local
VITE_WS_URL=ws://localhost:3000/ws
```

## 🔍 驗證連接

打開瀏覽器開發者工具 (F12)，查看 Console：

### 成功連接
```
[WebSocket] 連接到: wss://165.227.147.40:3000/ws
[WebSocket] 已連接
```

### 連接失敗
```
[WebSocket] 連接到: wss://165.227.147.40:3000/ws

```

## 📊 技術說明

### 為什麼會有這個問題？

1. **Netlify 前端**: 使用 HTTPS (有效證書)
2. **後端伺服器**: 使用 HTTPS (自簽名證書)
3. **瀏覽器安全策略**: 不信任自簽名證書

### 為什麼不能用 HTTP？

HTTPS 網站無法連接到 HTTP WebSocket (瀏覽器 Mixed Content 政策)

### 為什麼不用 Netlify 反向代理？

Netlify 免費方案不支援 WebSocket 反向代理

---

## 🎯 長期解決方案

### 選項 1：使用有效的 SSL 證書
使用 Let's Encrypt 為後端申請免費的有效證書

### 選項 2：使用 Cloudflare Tunnel
通過 Cloudflare Tunnel 提供有效的 HTTPS

### 選項 3：部署到同一域名
將前後端部署到同一域名下，避免跨域問題

---

## 📞 需要幫助？

如果按照上述步驟仍無法連接，請檢查：

1. ✅ 後端伺服器是否正常運行
2. ✅ 防火牆是否開放 port 3000
3. ✅ 瀏覽器是否已接受證書
4. ✅ 網路連接是否正常
