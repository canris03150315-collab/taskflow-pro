## 📋 功能需求確認

### 假別類型
- 病假 (SICK)
- 事假 (PERSONAL)
- 特休 (ANNUAL)
- 婚假 (MARRIAGE)
- 喪假 (BEREAVEMENT)
- 產假 (MATERNITY)
- 生理假 (MENSTRUAL)

### 核心流程
1. 員工排假申請
2. 系統自動衝突檢查
3. 有衝突 → 通知員工和主管
4. 無衝突 → 等待部門所有人排完假
5. 主管批次確認（可覆蓋衝突批准）

### 權限設定
- 部門主管：設定自己部門規則、審核自己部門假單
- BOSS：設定所有部門規則、查看所有部門假單

### 整合需求
- 與出勤系統整合（不影響已有數據）
- 使用企業通訊軟體（聊天系統）通知

## 📦 實現階段

### 階段 1：後端 API（Day 1-2）

#### 1.1 假期申請 API
- POST /api/leaves - 創建假期申請
- GET /api/leaves - 獲取假期列表
- GET /api/leaves/:id - 獲取單個假期
- PUT /api/leaves/:id - 更新假期申請
- DELETE /api/leaves/:id - 取消假期申請

#### 1.2 部門規則 API
- GET /api/leaves/rules/:departmentId - 獲取部門規則
- PUT /api/leaves/rules/:departmentId - 設定部門規則

#### 1.3 假期審核 API
- POST /api/leaves/:id/approve - 批准假期
- POST /api/leaves/:id/reject - 駁回假期
- POST /api/leaves/batch-approve - 批次批准

#### 1.4 假期額度 API
- GET /api/leaves/quotas/:userId - 獲取用戶假期額度
- PUT /api/leaves/quotas/:userId - 更新假期額度

#### 1.5 衝突檢查 API
- POST /api/leaves/check-conflict - 檢查衝突

### 階段 2：衝突檢查引擎（Day 2）

實現智能衝突檢查邏輯：
- 同時段請假人數檢查
- 最少在崗人數檢查
- 關鍵職位檢查
- 禁止日期檢查
- 提前天數檢查
- 連續天數檢查

### 階段 3：前端頁面（Day 3-4）

#### 3.1 主管規則設定頁面
- 部門排假規則設定表單
- 關鍵日期設定
- 關鍵職位設定

#### 3.2 員工排假申請頁面
- 互動式日期選擇
- 即時衝突檢查
- 智能建議
- 查看同時段請假人員

#### 3.3 主管批次確認頁面
- 排假行事曆視圖
- 批次確認列表
- 衝突警告
- 覆蓋衝突批准

#### 3.4 排假行事曆頁面
- 個人假期行事曆
- 部門假期行事曆
- 全公司假期行事曆（BOSS）

### 階段 4：通知與整合（Day 5）

#### 4.1 企業通訊軟體通知
- 假期申請通知
- 衝突警告通知
- 審核結果通知
- 假期提醒通知

#### 4.2 出勤系統整合
- 批准的假期自動標記在出勤記錄
- 不影響已有出勤數據
