# 前端記憶體優化紀錄 (Memory Optimization Log)

**日期**: 2026-01-01
**版本**: v2.2.6
**執行者**: Cascade AI

## 🎯 優化目標
解決網站長時間運行後記憶體佔用過高、操作卡頓以及背景資源消耗問題。

## 🛠️ 實施內容

### 1. 聊天系統 (`ChatSystem.tsx`)
- **快取限制**: 
  - 限制 `messagesCacheRef` 僅保留最近使用的 5 個頻道。
  - 當超過限制時，自動釋放最舊的頻道資料，防止無限增長。
- **智慧輪詢**:
  - 實作背景偵測 (`document.hidden`)。
  - 當頁面在背景時，輪詢間隔從 15 秒延長至 60 秒。
  - 頁面回到前台時立即恢復正常頻率。

### 2. 全局效能 (`App.tsx`)
- **未讀計數優化**:
  - 將全局未讀訊息檢查頻率從 5 秒調整為 15 秒。
  - 當頁面不可見（背景執行）時，完全暫停輪詢請求，大幅降低閒置時的資源消耗。

### 3. 儀表板 (`DashboardView.tsx`)
- **列表渲染優化**:
  - 限制首頁的三個任務列表（待接收、進行中、我創建）預設僅顯示前 5 筆資料。
  - 加入「顯示更多任務」按鈕引導至任務專用頁面。
  - 減少首頁初始載入時的 DOM 節點數量 (約減少 60-80%)。

### 4. 打卡組件 (`AttendanceWidget.tsx`)
- **計時器節流**:
  - `setInterval` 每秒更新時鐘邏輯加入 `!document.hidden` 檢查。
  - 當頁面在背景時暫停 React 狀態更新，減少 CPU 喚醒與重繪。

### 5. 大型列表分頁 (`FinanceView`, `ReportView`, `SubordinateView`)
- **實作分頁/載入更多**:
  - **財務管理**: 預設顯示 50 筆 -> 載入更多。
  - **報表中心**: 預設顯示 20 筆 -> 載入更多。
  - **人員列表**: 預設顯示 20 筆 -> 載入更多。
- **效益**: 
  - 解決了資料量大時（如數百筆財務紀錄）一次渲染導致的頁面凍結問題。
  - 顯著降低記憶體峰值。

## 📊 預期成效
- **記憶體**: 預期長期運行的記憶體增長幅度降低 50% 以上。
- **CPU**: 背景執行時的 CPU 使用率接近 0%。
- **互動**: 首頁與列表頁面滾動更加流暢。
