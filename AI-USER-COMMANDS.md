
```

```

## 2. AI 不遵守規則時

### 🎯 目的
立即糾正 AI 的錯誤行為，避免造成系統損壞。

### 🚨 警告信號（需要立即糾正）

如果 AI 出現以下行為，**立即使用糾正話術**：

- ❌ 使用過時的版本號（不是從 WORK_LOG_CURRENT.md 獲取）
- ❌ 提到使用 PowerShell `&&`
- ❌ 建議跳過備份
- ❌ 使用錯誤的 Netlify Site ID
- ❌ 建議直接部署到生產（不經測試）
- ❌ 說「我記得」但給出錯誤資訊
- ❌ 建議編譯 TypeScript
- ❌ 建議使用 scp 命令

### 📝 糾正話術（依嚴重程度）

#### 輕微違規（第一次）

```
⚠️ 等一下，你剛才的建議不正確。請重新閱讀 AI-MANDATORY-CHECKLIST.md 和 WORK_LOG_CURRENT.md，然後重新回答。
```

#### 中度違規（第二次或嚴重錯誤）

```
🔴 停止！你違反了核心規則。請立即：
1. 重新閱讀 AI-MANDATORY-CHECKLIST.md
2. 重新閱讀 WORK_LOG_CURRENT.md 前 50 行
3. 列出 5 個絕對禁止的操作
4. 確認當前系統版本

完成後再繼續。
```

#### 嚴重違規（第三次或可能造成損壞）

```
🚫 緊急停止！你已經多次違反規則。

請執行完整重置：
1. 閱讀 AI-MANDATORY-CHECKLIST.md
2. 閱讀 WORK_LOG_CURRENT.md（前 50 行）
3. 閱讀 PROJECT-QUICKSTART.md

然後回答以下問題：
1. 當前前端 Deploy ID 是什麼？
2. 當前後端版本是什麼？
3. PowerShell 可以使用 && 嗎？為什麼？
4. 修改代碼前必須做什麼？
5. 前端部署前必須做什麼？
6. 後端修改後忘記 docker commit 會怎樣？
7. 為什麼不能編譯 TypeScript？
8. 為什麼不能使用 scp 命令？

如果你無法正確回答所有問題，我會考慮更換 AI。
```

### 🔍 具體錯誤的糾正話術

#### 錯誤 1：使用過時版本號

```
❌ 你使用的版本號不正確。請從 WORK_LOG_CURRENT.md 第 12 行和第 20 行獲取最新的前端 Deploy ID 和後端版本。
```

#### 錯誤 2：建議使用 PowerShell &&

```
❌ PowerShell 不支援 && 運算符！請使用分號 ; 分隔命令。這是核心規則，請重新閱讀 AI-MANDATORY-CHECKLIST.md。
```

#### 錯誤 3：建議跳過備份

```
❌ 絕對不可以跳過備份！修改代碼前必須創建快照。請使用：
ssh root@165.227.147.40 "/root/create-snapshot.sh v版本號-before-修改描述"
```

#### 錯誤 4：使用錯誤的 Netlify Site ID

```
❌ Netlify Site ID 錯誤！
生產環境：5bb6a0c9-3186-4d11-b9be-07bdce7bf186
測試環境：480c7dd5-1159-4f1d-867a-0144272d1e0b
請確認你要部署到哪個環境。
```

#### 錯誤 5：建議編譯 TypeScript

```
❌ 不要編譯 TypeScript！後端直接修改 /app/dist 中的 JS 檔案。這是核心規則，請重新閱讀 AI-MANDATORY-CHECKLIST.md。
```

#### 錯誤 6：建議使用 scp

```
❌ 不要使用 scp 命令！請使用 Get-Content | ssh 管道上傳文件：
Get-Content "檔案.js" -Raw | ssh root@165.227.147.40 "cat > /tmp/檔案.js"
```

## 3. 結束一天工作時

### 🎯 目的
確保所有變更已保存，系統狀態已記錄，方便下次繼續工作。

### 📝 標準話術（複製貼上）

```
今天的工作要結束了，請幫我完成以下檢查：

1. 確認所有變更已 Git commit
2. 確認 WORK_LOG_CURRENT.md 已更新
3. 列出今天完成的工作
4. 列出當前系統狀態：
   - 前端 Deploy ID
   - 後端 Docker 映像版本
   - 最後更新時間
5. 列出未完成的工作（如果有）
6. 建議是否需要創建備份

完成後，給我一個簡潔的工作總結。
```

### ✅ AI 應該回應的內容

AI 應該提供：

```
✅ 結束工作檢查完成：

1. Git 狀態：
   - ✅ 所有變更已 commit
   - 最後 commit: [commit hash] - [描述]

2. WORK_LOG_CURRENT.md：
   - ✅ 已更新
   - 最後更新時間：YYYY-MM-DD HH:MM

3. 今天完成的工作：
   - [工作項目 1]
   - [工作項目 2]
   - [工作項目 3]

4. 當前系統狀態：
   - 前端 Deploy ID: 697210e41580c21f5b1e3092
   - 後端版本: taskflow-pro:v8.9.169-audit-db-syntax-fix
   - 最後更新：2026-01-22 19:58
   - 狀態：✅ 正常運行

5. 未完成的工作：
   - [待辦事項 1]（如果有）
   - [待辦事項 2]（如果有）

6. 備份建議：
   - ✅ 建議創建備份（今天有重要修改）
   或
   - ⚠️ 不需要備份（今天只有文檔更新）

📊 工作總結：
今天完成了 [X] 個任務，系統狀態正常。[簡要描述主要成果]。
```

### 🔧 如果需要創建備份

AI 建議創建備份時，使用：

```
好的，請幫我創建今天的備份。使用版本號 v8.9.170 和描述「2026-01-24 日常備份」。
```

AI 應該提供完整的備份命令。

## 4. 其他常用場景

### 場景 A：對話太長（超過 20 輪或 30 分鐘）

```
🔄 我們已經對話了很久，請執行完整重置以確保規則正確。

重新閱讀：
1. AI-MANDATORY-CHECKLIST.md
2. WORK_LOG_CURRENT.md（前 50 行）
3. PROJECT-QUICKSTART.md

然後確認當前系統狀態。
```

### 場景 B：切換任務

```
🔄 我們要開始新任務了，請先執行快速重置確認系統狀態。

重新閱讀 AI-MANDATORY-CHECKLIST.md 和 WORK_LOG_CURRENT.md 前 50 行，然後告訴我當前版本資訊。
```

### 場景 C：準備部署

```
📋 部署前檢查：

請確認：
1. 當前前端 Deploy ID
2. 當前後端版本
3. 前端部署的完整步驟（5 步）
4. 後端修改的完整步驟（7 步）
5. 是否已創建備份

確認完成後，我們再開始部署。
```

### 場景 D：快速版本確認

```
✅ 快速確認：當前前端 Deploy ID 和後端版本是什麼？
```

### 場景 E：規則檢查

```
⚠️ 規則檢查：請列出 5 個絕對禁止的操作和 5 個必須遵守的規則。
```

### 場景 F：發現系統異常

```
🚨 系統異常：[描述問題]

請先檢查：
1. 後端容器狀態
2. 前端訪問狀態
3. 最近的修改記錄（WORK_LOG_CURRENT.md）
4. 最新的備份

然後建議解決方案。
```

### 場景 G：需要回溯版本

```
⏮️ 需要回溯：[描述原因]

請告訴我：
1. 當前版本資訊
2. 可用的備份列表
3. 回溯步驟
4. 回溯風險

等我確認後再執行。
```

## 🎯 使用技巧

### 1. 複製貼上，不要手打

所有話術都設計成可以直接複製貼上，避免打錯字或遺漏重要部分。

### 2. 根據情況調整

如果 AI 只是輕微錯誤，使用輕度糾正話術。如果是嚴重違規，使用嚴重違規話術。

### 3. 定期重置

- 每 20 輪對話 → 重置
- 每 30 分鐘 → 重置
- 切換任務 → 重置
- 部署前 → 重置

### 4. 保持記錄

可以在筆記中記錄：
```
- [時間] 開啟新對話 - AI 確認完成檢查
- [時間] 糾正 AI - 原因：使用過時版本號
- [時間] 結束工作 - 完成 3 個任務
```

### 5. 不要省略步驟

即使 AI 說「我記得」或「我知道」，仍然要求它重新確認。記憶會衰減，重新確認更安全。

## 🚨 緊急情況處理

### 如果 AI 已經執行了錯誤操作

```
🚨 緊急停止！你剛才的操作可能有問題。

請立即：
1. 停止所有操作
2. 告訴我你剛才執行了什麼命令
3. 檢查系統當前狀態
4. 列出最近的備份

等我確認後再繼續。
```

### 如果系統出現異常

```
🚨 系統異常！

請執行緊急檢查：
1. 後端容器狀態：ssh root@165.227.147.40 "docker ps"
2. 後端日誌：ssh root@165.227.147.40 "docker logs taskflow-pro --tail 50"
3. 前端訪問測試
4. 最近的修改記錄

然後建議恢復方案。
```
