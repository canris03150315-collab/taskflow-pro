## 技術細節

### 當前備份方法
使用 `cp` 命令直接複製資料庫文件

**優點**：
- 簡單快速
- 容易實施

**缺點**：
- 可能在資料庫寫入時複製到不一致狀態
- 沒有使用 SQLite 官方推薦的方法

### 建議的改善方法
使用 SQLite BACKUP API（better-sqlite3 的 backup() 方法）

**優點**：
- 官方推薦的備份方法
- 即使資料庫正在使用也能安全備份
- 保證備份的一致性和完整性

**實施方法**：
```javascript
const Database = require('better-sqlite3');
const source = new Database('/app/data/taskflow.db', { readonly: true });
const backup = source.backup('/tmp/backup.db');
let remaining = -1;
while (remaining !== 0) {
  backup.step(100);
  remaining = backup.remainingPages;
}
backup.close();
source.close();
```

**檢查**：
1. 容器是否運行
2. 磁碟空間是否充足
3. 備份腳本權限是否正確

**解決**：
```bash
# 檢查容器狀態
docker ps | grep taskflow-pro

# 檢查磁碟空間
df -h

# 檢查腳本權限
ls -l /root/backup-taskflow.sh
```

### 備份數量異常
