# v2.0.2 修復說明

## 🐛 問題描述
- 新增員工後，數據不會即時顯示在列表中
- 新增部門後，數據不會即時顯示在列表中
- 需要重新整理頁面才能看到新添加的數據

## 🔍 問題根源
### 後端返回格式
後端 API 返回的數據格式是：
```json
{
  "user": { ... },
  "message": "用戶創建成功"
}
```

```json
{
  "department": { ... },
  "message": "部門創建成功"
}
```

### 前端處理問題
前端 API 服務直接返回了整個響應對象，而不是提取其中的 `user` 或 `department` 字段：

```typescript
// 錯誤的代碼
create: (user: User) => request<User>('POST', '/users', user),
create: (dept: DepartmentDef) => request<DepartmentDef>('POST', '/departments', dept),
```

## ✅ 修復方案
修改前端 API 服務，正確提取返回的數據對象：

```typescript
// 修復後的代碼
create: async (user: User) => {
    const response = await request<{ user: User, message: string }>('POST', '/users', user);
    return response.user;
},
create: async (dept: DepartmentDef) => {
    const response = await request<{ department: DepartmentDef, message: string }>('POST', '/departments', dept);
    return response.department;
},
```

## 📝 修復的文件
